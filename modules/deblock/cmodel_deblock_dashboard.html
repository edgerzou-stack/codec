    C-Model 去块滤波 (Deblock) 架构总结
    Architecture & Implementation Details

    1. 全局流水线架构

    Deblock 过程以 CTU 为基本处理单元。整体流水线如下：

    2. 数据准备与入口 (ctu_deblock)

    入口函数为 ctu_deblock，主要负责环境初始化与数据搬运。

    💡 核心机制：
    1. 像素拷贝：将重建缓存 (recon) 的像素完整拷贝到去块缓存 (deblock)。
    2. LMCS 支持：如果开启亮度的逆映射处理（sliceReshaperEnableFlag），在去块之前执行 rspSignal。
    3. 上下文构建：计算当前 CTU 的空间坐标（行、列）、边界尺寸（location_t loc），并分配 bs_t bS[256 * 2] 用来存储水平与垂直边界强度。

    3. 边界强度计算 (Boundary Strength)

    边界强度是滤波算法的核心控制信号，决定后续是否执行滤波以及滤波强度。具体在 bsd.c 的 bsd_calculate 中实现。

    graph LR A["检测 4x4 网格边缘"] --> B{"是否有边界?"} B -- Yes --> C{"是否 Intra 预测?"} C -- Yes --> D["BS = 2 或 3 (最高强度)"] C -- No --> E{"非零系数 (CBF)?"} E -- Yes --> F["BS = 1 或 2"] E -- No --> G{"MV 或 RefIdx 差异大?"} G -- Yes --> H["BS = 1"] G -- No --> I["BS = 0 (跳过滤波)"]
    图 2：边界强度 (BS) 判定状态机
    架构实现细节：
    bs_t 数组分为两部分：0~255 对应左侧垂直边界，256~511 对应上方水平边界。同时会针对 VVC 长抽头滤波需求，更新 maxFilterLengthP 和 maxFilterLengthQ。

    4. 亮度去块滤波 (Luma Deblock)

    亮度去块根据边缘方向调用 FilterVerY 和 FilterHorY。在 VVC 模式下，则调用对应的 _vvc 函数版本。

    4.1 参数推导与梯度检测

    基于计算得出的 BS 值与当前 QP，查表获取 tc（最大裁剪阈值）和 beta（平滑度阈值）。利用 calcD 函数计算边界两侧的像素梯度 d。

    💡 BS 与亮度滤波阈值的关系：
    1. 当 BS == 0 时，直接跳过滤波。
    2. 当 BS > 0 时，随着 BS 的增加（如 1 -> 2 -> 3），在查表计算 tc 时会产生正向的提升（公式中对应 + 2 * (BS - 1) 的偏移）。
    3. 结论：BS 强度越大，算出的 tc（最大修改阈值）越大，去块滤波在边界两侧允许“涂抹”的像素范围幅度也就越大，实际呈现的滤波效果更强。

    💡 VVC 长抽头扩展：
    当 maxFilterLengthP > 3 时，表明存在大 TU 或特殊划分（如 Affine），系统将触发长抽头强滤波，平滑范围最多可达 7 个像素（dbCoeffs7），以抑制极严重的块效应。

    4.3 核心滤波公式 (C-Model 实现)

    在理解滤波公式前，我们需要先明确像素在空间上的物理分布（以垂直边缘去块为例，p 为左侧块，q 为右侧块，数字表示距离边界的远近）：

    块 P (例如左侧)

    p[-3]
    p2
    p[-2]
    p1
    p[-1]
    p0
    p[0]
    边界
    块 Q (例如右侧)
    q0
    q[0]
    q1
    q[1]
    q2
    q[2]
    q3
    q[3]
    图 4：垂直边缘去块时，p 侧与 q 侧像素的物理空间映射与代码指针（数组索引）对应关系
    💡 指针 p[] 与 q[] 负索引含义：
    在 C-Model 的实现中，传入滤波函数的指针 p 默认指向 贴近边界的那个像素 p0。因为 P 块处于边界左侧，在数组（一维展开内存）中，越远离边界的像素（如 p1, p2）其物理内存地址越靠前。
    因此，代码利用负数数组索引来向左寻址：
    - p[0] = p0（紧贴边界）
    - p[-1] = p1（远离边界1个像素）
    - p[-2] = p2（远离边界2个像素）
    对于右侧的 Q 块，指针 q 同样指向 q0，由于它向右延伸，因此直接使用正向索引 q[0], q[1], q[2]。

    以下是基于上述物理像素排布真实执行的核心滤波公式：

    弱滤波 (Weak Filter)

    弱滤波影响范围示意：
    p3
    p2
    p1 (条件修改)
    p0 (必改)
    q0 (必改)
    q1 (条件修改)
    q2
    q3

    计算基础偏移量 delta，并限制在 [-tc, tc] 范围内。主要修改 p0, q0，如果满足额外平滑条件（分别判断左右），也会以一半的 tc 阈值修改 p1, q1：

    delta = (9 * (q0 - p0) - 3 * (q1 - p1) + 8) >> 4;
if (ABS(delta) < 10 * tc) {
    delta = CLIP3(-tc, tc, delta);
    p[0] = CLIP1(p0 + delta);
    q[0] = CLIP1(q0 - delta);
    
    // 如果满足 filter2P 条件，进一步修改 p1
    d1 = CLIP3(-(tc>>1), (tc>>1), (((p2 + p0 + 1) >> 1) - p1 + delta) >> 1);
    p[-1] = CLIP1(p1 + d1);
}

    强滤波 (Strong Filter - HEVC标准 3-tap)

    强滤波影响范围示意：
    p3 (参考)
    p2 (必改)
    p1 (必改)
    p0 (必改)
    q0 (必改)
    q1 (必改)
    q2 (必改)
    q3 (参考)

    直接通过两侧多个像素的加权平均来彻底平滑边界，并用 tc 乘以 2 进行最终的限幅（因为强滤波修改幅度更大）：

    // p 侧的 3 个像素平滑
p[0]  = CLIP3(p0 - 2*tc, p0 + 2*tc, (p2 + 2*p1 + 2*p0 + 2*q0 + q1 + 4) >> 3);
p[-1] = CLIP3(p1 - 2*tc, p1 + 2*tc, (p2 + p1 + p0 + q0 + 2) >> 2);
p[-2] = CLIP3(p2 - 2*tc, p2 + 2*tc, (2*p3 + 3*p2 + p1 + p0 + q0 + 4) >> 3);
// q 侧同理对称计算...

    5. 色度去块滤波 (Chroma Deblock)

    色度过程更加保守，调用 FilterVerC 和 FilterHorC。

    💡 BS 与色度滤波的激活关系：
    1. 当 BS < 2 时：默认跳过色度滤波。因为色度自身的频带较窄、图像更加平滑，且人眼对其块效应不如对亮度敏感。
    2. 当 BS >= 2 时：激活色度滤波，此时会根据色度专属的 QP 独立推导出 tc 并执行两端平滑。
    3. VVC 长抽头特例：如果边界触发了 VVC 的大区块长抽头滤波条件 (largeBoundary)，则对色度放宽条件，即使 BS == 1 也会强行激活滤波。

    5.1 色度滤波公式

    色度滤波影响范围示意：
    p3
    p2
    p1 (仅参考)
    p0 (必改)
    q0 (必改)
    q1 (仅参考)
    q2
    q3

    色度的计算只有弱滤波逻辑（除非触发 VVC 的大区块长抽头），计算时使用 p1 和 q1 作为辅助参考，但最终仅修改边界紧邻的 1 个像素：

    delta = CLIP3(-tc, tc, ((((q0 - p0) << 2) + p1 - q1 + 4) >> 3));
p[0] = CLIP1(p0 + delta);
q[0] = CLIP1(q0 - delta);

    6. 输出写回 (Store Output)

    经过处理的像素保存在 deblock 缓存中。通过 FilterOutput，系统按照左侧边缘、上方边缘、以及中心区域的三块划分，精准地将像素拼接覆盖回 p->deblock.lum.pel 等全局存储结构中，为后续的 SAO / ALF 模块准备数据。
